Module Revive  

This module gives the possibility to send and receive revive calls mentionning @Reviver between throughout Discord servers. The destination servers can be configured by the server admin. The bot can potentially send calls to servers that host the YATA bot and have their revive module activated.
Disclamer: The bot does not handle the revive payments or contracts.

Command !revive  

Sends a revive calls to the current server and to other servers (depending on your server's configuration). A revive call mention the role @Reviver and has the format:

@Reviver Kivou [2000607] from Nub Navy needs a revive https://www.torn.com/profiles.php?XID=2000607
In hospital for 3h28min (hospitalized by Ched)

If <tornId> is given it sends the call for player ID <tornId> on behalf of the author.

All messages of a call are deleted from the servers after 5 minutes.

Syntax: !revive or !revive <tornId>

Aliases: !r

Scope: Anyone can use it in #revive.

Command !reviver  

Gives or removes the @Reviver role of the command author.

Syntax: !reviver

Scope: Anyone can use it anywhere.

Command !reviveServers  

Allows server admin to set the different servers linked with them. They can set up two lists:

  • Server list: the list of servers to which the calls will be sent.
  • Black list: the list of servers from which the calls will be blocked (in case a server added you without your consent).

Syntax: !reviveServers or !reviveServers <discordServerId> or !reviveServers <-discordServerId>

Aliases: !rs

  • !reviveServers list all the servers with the current settings (discordServerId/contact/send/receive/blacklist).
  • !reviveServers <discordServerId> add the server to your server list.
  • !reviveServers <-discordServerId> add the server to your black list.

Scope: Server admin can use it in #yata-admin.
